# RLM Pitfall Ablation — Phase 1 Design Spec

## Goal

Reduce two stylistic pitfalls (verbosity = 31 trips; boilerplate openings = 12 trips) in RLM answers by tweaking the `papers_qa` system prompt and lowering the sampling temperature. Measure the delta with a sidecar v2 run + diff report so any regression in substantive scoring is visible.

## Non-goals

- **Not** addressing substantive failure modes (comparative_balance 52.6% miss, mechanism_explanation 27.5%, factual_anchor 26.1%) — those need a different design pass after Phase 1's effect is measured.
- **Not** fixing the 11 truncated `"给出多个"` rubric criteria — that's a bug in the rubric-generation pipeline, separate concern.
- **Not** swapping the candidate model. Stays on `deepseek/deepseek-v4-flash`.
- **Not** overwriting any existing v1 files. v1 must remain intact for A/B comparison.

## Inputs

- `data/CAE-v2.0-1-rubrics.json` — 94 items with v1 `rlm_answer` and full criteria (read-only).
- `outputs/scoring/cae-v2.0-1-scores.json` — v1 per-item scoring results (read-only).
- `data/CAE-anchor-scores.json` — cached anchors (read-only, reused).
- `papers_qa/papers_qa/prompts.py` — `BILINGUAL_ADDENDUM` template (modified).
- `papers_qa/.env` — `PAPERS_QA_TEMPERATURE` value (modified).

## Outputs

- **`papers_qa/papers_qa/prompts.py`** (modified): 3 new directives appended to `BILINGUAL_ADDENDUM`.
- **`papers_qa/.env`** (modified): `PAPERS_QA_TEMPERATURE=0.3` (was 0.8).
- **`data/CAE-v2.0-1-rubrics-v2.json`** — sidecar: rubrics with v2 `rlm_answer` (preserves v1 file intact). Same 94 items, same `item_idx`s. Generated by re-using the existing `src/generate_rlm_answers.py` with `--input data/CAE-v2.0-1-rubrics.json --output data/CAE-v2.0-1-rubrics-v2.json --jsonl outputs/rlm-answers/cae-v2.0-1-v2.jsonl`.
- **`outputs/scoring/cae-v2.0-1-scores-v2.json`** and **`cae-v2.0-1-report-v2.md`** — produced by `src/score_rlm_answers.py` with `--input data/CAE-v2.0-1-rubrics-v2.json --scores-out outputs/scoring/cae-v2.0-1-scores-v2.json --report-out outputs/scoring/cae-v2.0-1-report-v2.md`.
- **`src/rubrics_diff_report.py`** (new, ~150 lines, pure-function): given two scores.json files, render a Chinese markdown diff report.
- **`outputs/scoring/cae-v2.0-1-diff-v1-v2.md`** — v1-vs-v2 diff (per-item score deltas, pitfall trip deltas per text, by-criterion-type deltas, top-5 winners, top-5 losers).

## Prompt change (exact text)

Append to `BILINGUAL_ADDENDUM` in `papers_qa/papers_qa/prompts.py`, right before the final `=========` divider (if any) or at the end:

```
**回答风格要求（必须遵守）：**

1. **直接给出答案。** 不要以"好的"、"我们来回答"、"接下来我将…"、"以下是…的回答"
   等套话、开场白或元评论开头。第一句话就要是实质内容。
2. **紧凑、聚焦。** 答案应直接针对问题，不要展开与问题无直接关联的背景知识、
   相邻概念或重复内容。一段说清楚的就不要分三段说。
3. **不要给出多个相互矛盾的候选答案。** 例如 "可能是 A 也可能是 B" / "以下是
   几种可能的解释……"。选定一个最佳答案并直接给出；如确有多种合理理解，
   只在末尾用一句话标注即可。
```

## Temperature change

`papers_qa/.env`:
```
- PAPERS_QA_TEMPERATURE=0.8
+ PAPERS_QA_TEMPERATURE=0.3
```

Per `.env.example`: "0.7+ tends to hallucinate numbers on reasoning models". 0.3 is in the recommended safe zone (above the 0.0 strict-zero deadlock, below the hallucination threshold).

## Diff report sections

`outputs/scoring/cae-v2.0-1-diff-v1-v2.md`:

1. **Header** — v1 path, v2 path, generated_at, item count.
2. **总体得分对比** — mean raw + mean anchored, before / after / Δ.
3. **Pitfall 触发对比** — per pitfall text: v1 trips, v2 trips, Δ.
4. **按 criterion_type 命中率对比** — per criterion_type: v1 rate, v2 rate, Δ percentage points.
5. **按题型分组对比** — per question_type: v1 mean anchored, v2 mean anchored, Δ.
6. **Top 5 winners** — items where v2 anchored − v1 anchored is most positive.
7. **Top 5 losers** — items where v2 anchored − v1 anchored is most negative. Includes question text + the criteria that flipped (✅ → ❌).

## Pipeline reuse

No new orchestration code. The existing CLI tools support everything via flags:

```bash
# 1. Edit prompts.py and .env (manual)
# 2. Re-run RLM answers to sidecar file
.venv/bin/python src/generate_rlm_answers.py \
    --input data/CAE-v2.0-1-rubrics.json \
    --output data/CAE-v2.0-1-rubrics-v2.json \
    --jsonl outputs/rlm-answers/cae-v2.0-1-v2.jsonl \
    --papers-dir /home/juli/RLM/CAE-MDs \
    --workers 4

# 3. Re-score v2 answers
.venv/bin/python src/score_rlm_answers.py \
    --input data/CAE-v2.0-1-rubrics-v2.json \
    --scores-out outputs/scoring/cae-v2.0-1-scores-v2.json \
    --report-out outputs/scoring/cae-v2.0-1-report-v2.md \
    --judge-model openai/gpt-5.5

# 4. Generate diff report (new tool)
.venv/bin/python src/rubrics_diff_report.py \
    --scores-v1 outputs/scoring/cae-v2.0-1-scores.json \
    --scores-v2 outputs/scoring/cae-v2.0-1-scores-v2.json \
    --rubrics   data/CAE-v2.0-1-rubrics-v2.json \
    --out       outputs/scoring/cae-v2.0-1-diff-v1-v2.md
```

## Modules

- **`src/rubrics_diff_report.py`** (~150 lines) — `render_diff(scores_v1, scores_v2, rubrics, meta) -> str`, plus a `main()` with argparse. No I/O inside `render_diff` (pure function for testing). All file I/O in `main()`.
- **`tests/test_rubrics_diff_report.py`** (~10 tests) — overall delta math, per-pitfall delta, by-criterion-type delta, top-N winners/losers ordering, missing-item handling (item in v1 but not v2 → flagged), no-LLM.

## Pipeline data shapes

`scores.json` schema (from `src/rubrics/scorer.py` — re-used):
- `item_idx: int`, `score: float | None`, `score_anchored: {normalized: float|None}`, `breakdown: [{criterion_type, met, sign, text, ...}]`, `question_type: str`, `difficulty: str`.

`render_diff(scores_v1, scores_v2, rubrics, meta)` operates on these lists directly; no need to call `build_aggregate` (we compute deltas, not aggregates).

## Failure modes

| Failure | Handling |
|---|---|
| v2 run partially completes, then crashes | `generate_rlm_answers.py` resume already covers this — re-run picks up where it left off |
| v2 scoring partially completes | `score_rlm_answers.py` resume covers this |
| v2 score file has different `item_idx`s than v1 | `render_diff` reports `"items in v1 but not v2"` and `"in v2 but not v1"` counts at the top; computes deltas only on the intersection |
| Prompt change makes answers shorter but content-poor (anchored drops > 0.02) | Acceptance criterion fails; we revert the temperature change first (it's a single env-var revert), re-run, re-score |

## Acceptance criteria

1. Pitfall #1 (冗长) trip count drops from **31 → < 10** on v2.
2. Pitfall #2 (套话开场) trip count drops from **12 → < 3** on v2.
3. Overall mean anchored score does NOT decrease by more than 0.02 (v1 was 0.70 → v2 must be ≥ 0.68).
4. `src/rubrics_diff_report.py` has ≥ 10 tests, all passing.
5. The diff report `outputs/scoring/cae-v2.0-1-diff-v1-v2.md` is generated, lists ≥ 5 winners and ≥ 5 losers with their flipped criteria.
6. v1 files remain bit-for-bit identical (no overwrites).

## Risks

| Risk | Severity | Mitigation |
|---|---|---|
| Re-run cost (~$10) | LOW | Bounded by existing per-call budget cap |
| Re-score cost (~$15) | LOW | Same |
| Anchored regression > 0.02 | MEDIUM | AC #3; revert temperature first, re-measure |
| Prompt directive ignored by deepseek-v4-flash | MEDIUM | If pitfall trips don't drop, model is the bottleneck — escalate to a different model in a follow-up spec |
| Judge gpt-5.5 unavailable | LOW | Already battle-tested in the prior scoring run; same proxy |
| 一致性: same `criteria` in rubrics-v2.json as v1 | MEDIUM | `generate_rlm_answers.py` only writes `rlm_answer` + `rlm_error`; criteria fields are preserved untouched. Verify by `diff <(jq -S 'del(.[].rlm_answer, .[].rlm_error)' v1) <(jq -S 'del(.[].rlm_answer, .[].rlm_error)' v2)` showing zero differences. |

## What success enables

If AC pass, write a Phase 2 spec to attack the substantive failure modes (comparative_balance, mechanism_explanation, factual_anchor). Methodology proven; Phase 2 just changes the prompt directives and re-runs the same loop.

If AC fail, revert and pick a different lever (model swap, prompt restructure, or directly prompt-engineer one criterion_type at a time).
