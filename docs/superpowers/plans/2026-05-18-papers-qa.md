> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a thin wrapper around `rlm` (Recursive Language Models) that lets the user ask Chinese questions over a local corpus of 79 English markdown papers, using `gpt-5.4-mini` via aiberm.com as the backend, with a CLI + batch interface and a bilingual keyword search tool.

**Architecture:** The wrapper lives at `/home/juli/RLM/papers_qa/` as a sibling of `rlm/`. We load all 79 papers as a `dict[paper_id, str]` and pass it as the `prompt` argument to `RLM.completion()` (which becomes the `context` variable inside the REPL). The user's Chinese question is passed as `root_prompt`. A custom `search_papers()` tool is injected into the REPL so the in-REPL LLM can do fast case-insensitive multi-keyword substring search over English text. The RLM system prompt is augmented with bilingual instructions telling the model to translate Chinese keywords into English variants before searching, and to answer in Chinese.

**Tech Stack:**
- Python 3.11+, `uv`
- `rlms` (local editable install from `../rlm`)
- `python-dotenv` for `.env`
- `pytest` for tests
- No vector DB, no embeddings — pure substring search + LLM intelligence

**Why not RAG/embeddings?** RLM's whole point is the LLM writes its own retrieval code in a REPL. A vector index adds dependency surface for a 7.2MB corpus where grep is fast enough. Bilingual handling is solved at the prompt layer: the model translates Chinese terms before searching.

## How it works

1. All `*.md` files under `PAPERS_QA_PAPERS_DIR` are loaded into a `dict[paper_id, text]` (paper_id is filename without `.md`).
2. The dict is passed to `RLM.completion(prompt=papers_dict, root_prompt=user_question)` — RLM stores it as the `context` variable inside its Python REPL.
3. The in-REPL LLM (gpt-5.4-mini) sees a bilingual system prompt telling it: "questions are Chinese, papers are English; before grepping, translate keywords; you have a `search_papers()` tool".
4. The model iterates: search → chunk → llm_query sub-calls → aggregate → final answer in Chinese.

Trajectories are saved per question under `outputs/logs/` for the RLM visualizer.

# papers_qa/config.py
"""Frozen configuration for papers_qa, sourced from environment variables."""
from __future__ import annotations

import os
from dataclasses import dataclass
from pathlib import Path

from dotenv import load_dotenv


@dataclass(frozen=True)
class PapersQAConfig:
    """All runtime knobs in one immutable bundle."""

    api_key: str
    base_url: str
    model_name: str
    papers_dir: Path
    max_iterations: int
    max_depth: int
    max_budget_usd: float | None
    max_timeout_s: float | None
    log_dir: Path

    @classmethod
    def from_env(cls, env_file: str | None = None) -> "PapersQAConfig":
        """Build config from environment. Loads .env if present."""
        load_dotenv(env_file)  # no-op if file missing

        api_key = os.environ.get("OPENAI_API_KEY")
        if not api_key:
            raise ValueError(
                "OPENAI_API_KEY is required. Set it in .env or your shell."
            )

        papers_dir_str = os.environ.get("PAPERS_QA_PAPERS_DIR")
        if not papers_dir_str:
            raise ValueError("PAPERS_QA_PAPERS_DIR is required.")
        papers_dir = Path(papers_dir_str).expanduser().resolve()

        log_dir = Path(
            os.environ.get("PAPERS_QA_LOG_DIR", "outputs/logs")
        ).expanduser().resolve()

        return cls(
            api_key=api_key,
            base_url=os.environ.get("OPENAI_BASE_URL", "https://api.openai.com/v1"),
            model_name=os.environ.get("PAPERS_QA_MODEL", "gpt-5.4-mini"),
            papers_dir=papers_dir,
            max_iterations=int(os.environ.get("PAPERS_QA_MAX_ITERATIONS", "30")),
            max_depth=int(os.environ.get("PAPERS_QA_MAX_DEPTH", "2")),
            max_budget_usd=_optional_float("PAPERS_QA_MAX_BUDGET_USD"),
            max_timeout_s=_optional_float("PAPERS_QA_MAX_TIMEOUT_S"),
            log_dir=log_dir,
        )


def _optional_float(key: str) -> float | None:
    raw = os.environ.get(key)
    if raw is None or raw == "":
        return None
    return float(raw)
```

# papers_qa/loader.py
"""Load English markdown papers from a directory into a dict keyed by filename stem."""
from __future__ import annotations

import logging
from pathlib import Path

logger = logging.getLogger(__name__)


def load_papers(papers_dir: Path) -> dict[str, str]:
    """Read every *.md file under ``papers_dir`` into a {filename_stem: full_text} dict.

    Args:
        papers_dir: Directory containing flat list of .md files.

    Returns:
        Dict mapping paper_id (filename without .md) to full text.

    Raises:
        FileNotFoundError: if directory does not exist or contains no .md files.
    """
    papers_dir = Path(papers_dir)
    if not papers_dir.exists():
        raise FileNotFoundError(f"Papers directory not found: {papers_dir}")

    md_files = sorted(papers_dir.glob("*.md"))
    if not md_files:
        raise FileNotFoundError(f"No .md files in {papers_dir}")

    papers: dict[str, str] = {}
    for path in md_files:
        papers[path.stem] = path.read_text(encoding="utf-8")
    logger.info("Loaded %d papers from %s", len(papers), papers_dir)
    return papers
```

# papers_qa/search.py
"""Custom search tool injected into the RLM REPL.

The in-REPL LLM is bilingual: it receives a Chinese question, expands it into
English keyword variants (synonyms, abbreviations, expansions), and calls
`search_papers(keywords=[...])` to find candidate papers/snippets fast.
"""
from __future__ import annotations

import re
from typing import Any

SEARCH_DESCRIPTION = (
    "search_papers(keywords: list[str], top_k=30, snippet_chars=400, "
    "paper_ids=None) -> list[dict]. Case-insensitive substring search across "
    "all loaded English papers. Returns the top_k papers by total keyword hit "
    "count (OR over keywords), each as {paper_id, score, snippet, hits}. "
    "Use this for fast keyword lookup BEFORE running expensive llm_query calls "
    "over full paper text. Pass multiple English synonyms / acronyms / expanded "
    "forms (the user's question is likely Chinese — translate first)."
)


def build_search_tool(papers: dict[str, str]) -> dict[str, Any]:
    """Return a custom_tools dict registering ``search_papers`` against ``papers``.

    The closure captures ``papers`` by reference; do not mutate it after build.
    """
    # Pre-lower for case-insensitive matching. Done once at build time.
    lowered: dict[str, str] = {pid: text.lower() for pid, text in papers.items()}

    def search_papers(
        keywords: list[str],
        top_k: int = 30,
        snippet_chars: int = 400,
        paper_ids: list[str] | None = None,
    ) -> list[dict[str, Any]]:
        if not keywords:
            return []
        kws_lower = [k.lower() for k in keywords if k]
        if not kws_lower:
            return []

        candidate_ids = paper_ids if paper_ids is not None else list(papers.keys())

        scored: list[dict[str, Any]] = []
        for pid in candidate_ids:
            text_lower = lowered.get(pid)
            if text_lower is None:
                continue
            hits: dict[str, int] = {}
            first_pos: int | None = None
            for kw in kws_lower:
                count = text_lower.count(kw)
                if count > 0:
                    hits[kw] = count
                    pos = text_lower.find(kw)
                    if first_pos is None or pos < first_pos:
                        first_pos = pos
            if not hits:
                continue
            snippet = _extract_snippet(papers[pid], first_pos or 0, snippet_chars)
            scored.append(
                {
                    "paper_id": pid,
                    "score": sum(hits.values()),
                    "hits": hits,
                    "snippet": snippet,
                }
            )

        scored.sort(key=lambda r: (-r["score"], r["paper_id"]))
        return scored[:top_k]

    return {
        "search_papers": {
            "tool": search_papers,
            "description": SEARCH_DESCRIPTION,
        }
    }


def _extract_snippet(text: str, center: int, total_chars: int) -> str:
    """Slice ``total_chars`` of context centered around ``center``."""
    half = total_chars // 2
    start = max(0, center - half)
    end = min(len(text), center + half)
    prefix = "…" if start > 0 else ""
    suffix = "…" if end < len(text) else ""
    # Collapse runs of whitespace so snippets render compactly to the LLM.
    snippet = re.sub(r"\s+", " ", text[start:end]).strip()
    return f"{prefix}{snippet}{suffix}"
```

from rlm.utils.prompts import RLM_SYSTEM_PROMPT

BILINGUAL_ADDENDUM = """

===================== Bilingual Q&A Guide =====================

You are answering questions about a corpus of 79 English machine-learning research papers.

**Context shape:** The `context` variable is a Python dict:
    context: dict[str, str]   # {paper_id: full_paper_markdown_text}

Paper IDs look like `Agarwal_2024_2306.13649`, `DeepSeekAI_2025_2501.12948`, etc.
List all paper IDs with `list(context.keys())`. Each value is the full English
markdown of one paper (typically 30k-300k chars).

**Critical bilingual rule:** The user's question is almost always in CHINESE,
but the papers are in ENGLISH. Before searching, you MUST translate the user's
Chinese terms into multiple English variants. Generate 5-15 keyword candidates
covering:
  - direct translations (知识蒸馏 → "knowledge distillation")
  - common abbreviations (KD, GKD, RLHF, SFT, MoE, CoT, ...)
  - synonyms / related terms (蒸馏 → distill, distillation, distilling)
  - sub-concepts the question implies (on-policy, off-policy, teacher, student)
  - author / paper IDs if mentioned

Do NOT search English papers with Chinese keywords — they will not match.

**Recommended workflow:**

1. Translate the Chinese question; brainstorm an English-keyword list.
2. Call `search_papers(keywords=[...], top_k=10)` to find candidate papers.
3. For each high-scoring paper, either:
   (a) call `llm_query(f"Question: {q}\\nPaper text:\\n{context[pid]}")`
       (single-paper Q&A; the sub-LLM handles ~500k chars), or
   (b) call `llm_query_batched([...])` over multiple candidate papers at once
       for parallel extraction.
4. If a single paper is huge, chunk `context[pid]` first.
5. Aggregate findings; produce the final answer **in Chinese** unless the
   question explicitly requests English.

**Examples of good keyword expansion:**
  Chinese question: "GKD 跟 SeqKD 在损失函数上的区别"
  → keywords=["GKD", "Generalized Knowledge Distillation", "SeqKD",
              "Sequence-Level KD", "loss function", "JSD", "KL divergence",
              "forward KL", "reverse KL"]

  Chinese question: "MoE 模型在 long context 上的表现怎么样？"
  → keywords=["MoE", "Mixture of Experts", "long context",
              "long-context", "sparse", "routing", "experts"]

**Final answer language:** Respond in Chinese unless the user's question is
already in English. Cite paper IDs you used (e.g., "据 Agarwal_2024_2306.13649
所述...").

================================================================
"""


def build_bilingual_system_prompt() -> str:
    """Return RLM_SYSTEM_PROMPT extended with bilingual + corpus-specific guidance.

    The `{custom_tools_section}` placeholder is preserved so RLM's
    `build_rlm_system_prompt` can still `.format()` it.

# papers_qa/runner.py
"""High-level driver: load papers, build tool, build prompt, run RLM."""
from __future__ import annotations

import logging
from dataclasses import dataclass
from typing import Any

from rlm import RLM
from rlm.logger import RLMLogger

from papers_qa.config import PapersQAConfig
from papers_qa.loader import load_papers
from papers_qa.prompts import build_bilingual_system_prompt
from papers_qa.search import build_search_tool

logger = logging.getLogger(__name__)


@dataclass
class AskResult:
    question: str
    answer: str
    cost_usd: float | None
    duration_s: float
    trajectory: dict[str, Any] | None


class PapersQA:
    """Chinese-question / English-paper QA driver wrapping RLM."""

    def __init__(self, config: PapersQAConfig) -> None:
        self.config = config
        self.papers: dict[str, str] = load_papers(config.papers_dir)
        self.custom_tools: dict[str, Any] = build_search_tool(self.papers)
        self.system_prompt: str = build_bilingual_system_prompt()
        self.logger: RLMLogger = RLMLogger(log_dir=str(config.log_dir))
        self.rlm: RLM = RLM(
            backend="openai",
            backend_kwargs={
                "model_name": config.model_name,
                "api_key": config.api_key,
                "base_url": config.base_url,
            },
            environment="local",
            max_iterations=config.max_iterations,
            max_depth=config.max_depth,
            max_budget=config.max_budget_usd,
            max_timeout=config.max_timeout_s,
            custom_system_prompt=self.system_prompt,
            custom_tools=self.custom_tools,
            logger=self.logger,
            verbose=True,
        )
        logger.info(
            "PapersQA ready: %d papers, model=%s, base_url=%s",
            len(self.papers),
            config.model_name,
            config.base_url,
        )

    def ask(self, question: str) -> AskResult:
        """Ask a (Chinese) question against the corpus, return the answer + trajectory."""
        completion = self.rlm.completion(prompt=self.papers, root_prompt=question)
        cost = (
            completion.usage_summary.total_cost
            if completion.usage_summary
            else None
        )
        return AskResult(
            question=question,
            answer=completion.response,
            cost_usd=cost,
            duration_s=completion.execution_time,
            trajectory=completion.metadata,
        )
```

```python
# scripts/ask_batch.py
"""Batch-answer a JSONL of questions.

Input JSONL (one question per line):
    {"id": "q1", "question": "你的问题"}

Output JSONL (one answer per line):
    {"id": "q1", "question": "...", "answer": "...", "cost_usd": 0.01,
     "duration_s": 1.2, "trajectory_path": "outputs/logs/q1.jsonl",
     "error": null}

Errors are caught per-question so a single failure doesn't stop the batch.
"""
from __future__ import annotations

import argparse
import json
import logging
import sys
import traceback
from pathlib import Path

from papers_qa.config import PapersQAConfig
from papers_qa.runner import PapersQA


def main() -> int:
    parser = argparse.ArgumentParser(description="Batch-answer Chinese questions.")
    parser.add_argument("input", type=Path, help="Input JSONL of {id, question}.")
    parser.add_argument("--out", type=Path, required=True, help="Output JSONL path.")
    parser.add_argument("--env-file", default=None)
    parser.add_argument("--log-level", default="INFO")
    args = parser.parse_args()

    logging.basicConfig(
        level=getattr(logging, args.log_level),
        format="%(asctime)s %(name)s %(levelname)s: %(message)s",
    )
    log = logging.getLogger("ask_batch")

    cfg = PapersQAConfig.from_env(args.env_file)
    qa = PapersQA(cfg)

    args.out.parent.mkdir(parents=True, exist_ok=True)
    with args.input.open() as fin, args.out.open("w") as fout:
        for lineno, raw in enumerate(fin, 1):
            raw = raw.strip()
            if not raw:
                continue
            try:
                row = json.loads(raw)
                qid = row["id"]
                question = row["question"]
            except (json.JSONDecodeError, KeyError) as e:
                log.error("Bad input on line %d: %s", lineno, e)
                continue

            log.info("[%s] %s", qid, question)
            record: dict = {"id": qid, "question": question}
            try:
                result = qa.ask(question)
                record["answer"] = result.answer
                record["cost_usd"] = result.cost_usd
                record["duration_s"] = result.duration_s
                record["error"] = None
            except Exception as e:
                log.exception("[%s] failed", qid)
                record["answer"] = None
                record["error"] = f"{type(e).__name__}: {e}"
                record["traceback"] = traceback.format_exc()

            fout.write(json.dumps(record, ensure_ascii=False) + "\n")
            fout.flush()

    log.info("Done. Wrote %s", args.out)
    return 0


if __name__ == "__main__":
    sys.exit(main())
```

## Risks

| Severity | Risk | Mitigation |
|----------|------|------------|
| HIGH | aiberm.com endpoint returns errors / rate-limits / non-OpenAI-spec responses | The `openai` client backend will surface this clearly; if it breaks, swap base_url to dubrify.com in `.env` (no code change). |
| HIGH | First load of 7MB dict into REPL via JSON serialization is slow (~1-2s) and may bloat the LLM's context-metadata view (79 chunk lengths) | Acceptable on first iteration; metadata prompt truncates at 100 chunks. If perf bad, switch to passing only paper_ids in context and lazy-loading text via a custom tool. |
| MEDIUM | LLM may try to dump the full 7MB context into a single llm_query (truncated) and produce hallucinations | Bilingual prompt explicitly tells it to use `search_papers` first, then chunk. |
| MEDIUM | Sub-LLM `llm_query` cost compounds — 79 papers × full-text per question is expensive | `max_budget_usd` and `max_iterations` limits set; default $2/question, 30 iter. |
| MEDIUM | Test_real_smoke and end-to-end checks burn real tokens on every CI run | `test_real_smoke` is gated by `RUN_RLM_SMOKE=1` env var; CI off by default. |
| LOW | `LocalREPL` writes context JSON to a tempdir which holds the full 7MB on every `completion()` call | Negligible (~7MB of disk per question, cleaned up on completion). |
| LOW | Paper ID collisions if duplicate filenames added | `load_papers` uses sorted filenames; would overwrite. Acceptable for current flat corpus. |

## Complexity: LOW-MEDIUM

- Loader + search + prompt: each ~50 lines, tested.
- Runner: ~80 lines, all glue.
- CLI shells: ~50 lines each.
- Total: ~500 lines of code + tests.
- End-to-end: 1-2 hours dev, plus iteration on prompt quality.
